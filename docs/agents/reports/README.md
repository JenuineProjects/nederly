# Pipeline Reports

One report per pipeline run, written by the Manager after Gate 3 (final) passes.

## Naming

`YYYY-MM-DD-NN-report.md`

Where `NN` is the next sequential number for that day (`01`, `02`, ...).

## Contents

Each report follows the structure defined in `../briefs/manager.md` → "Final report — produce after Gate 3 passes":

1. Run metadata
2. Researcher findings — 5-line summary
3. Gate decisions
4. Agent outputs produced
5. App changes made
6. Files touched
7. Open questions / deferred work
8. Lessons for next run

## Reading

The most recent report is the source of truth for *what changed in the last run*. Use `../manager_memory.md` as the cross-run index.
